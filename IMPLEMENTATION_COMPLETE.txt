ğŸ”‘ KEY FEATURES

TOKEN GENERATION
  When: User registers or logs in
  How: $user->createToken('auth_token')->plainTextToken
  Where: personal_access_tokens table
  Format: "1|abc123xyz..." (ID|Hash)

TOKEN VALIDATION
  When: Request to protected route
  How: auth:sanctum middleware
  Steps:
    1. Extract token from Authorization header
    2. Look up token in personal_access_tokens table
    3. Verify token matches hashed value
    4. Check token not expired
    5. Identify user from tokenable_id
    6. Set $request->user() to authenticated user

AUTHORIZATION
  When: Protected route or $this->authorize() called
  How: Policies (UserPolicy, ProfilePolicy)
  Check: Does user own the resource?
  Result: Allow or deny with 403 Forbidden

EMAIL VERIFICATION
  When: User registers
  How: Signed URL with SHA1 hash
  Check: User must verify before login
  Result: 403 if unverified, allow if verified

PASSWORD RESET
  When: User requests password reset
  How: One-time reset token sent via email
  Check: Token must be valid and not expired
  Result: Password updated after verification
